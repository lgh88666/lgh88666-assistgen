# AssistGen V3 Memory Architecture

## Goal

AssistGen V3 focuses on memory and context management for multi-turn ecommerce guidance.

The goal is not to build a heavy general-purpose memory platform. The goal is to make the shopping task state stable and explainable across turns.

## Current Status

V3 first implementation is complete:

- Redis-first session memory
- `InMemorySessionStore` fallback
- frontend `session_id` stored in `localStorage`
- `shopping_state` merge
- `effective_query`
- per-agent memory views

Deferred:

- LLM structured compression
- frontend context-compression running status
- visible new-chat button wiring

## Layer Position

Memory is a layer before the Agent chain, not a standalone Agent.

```text
User Query
  -> Memory Layer
  -> Query Understanding
  -> Supervisor
  -> Retrieval
  -> Recommendation
  -> Explanation
  -> Critic
  -> Final Answer
```

## Runtime Memory Store

```text
RedisSessionStore preferred
InMemorySessionStore fallback
```

Redis is not a blocking dependency for local development. If Docker/Redis is unavailable, the system continues with in-process session memory.

Suggested Redis keys:

```text
assistgen:session:{session_id}:messages
assistgen:session:{session_id}:shopping_state
assistgen:session:{session_id}:summary
```

Default TTL: 6 hours.

## Session ID

`session_id` identifies one chat session.

It should be generated by the frontend, stored in `localStorage`, and sent with every chat request. Refreshing the page keeps the same context. Starting a new chat creates a new `session_id`.

`user_id` should not be used as the memory key because one user can have multiple independent chat sessions.

## Shopping State

`shopping_state` is the structured business memory for the current shopping task.

Example:

```json
{
  "active_category": "robot_vacuum",
  "product_categories": ["robot_vacuum"],
  "budget_max": 1000,
  "ranking_objective": "best_value",
  "scenario": "home_cleaning",
  "preferred_brands": ["Mijia"],
  "rejected_brands": [],
  "hard_constraints": ["budget around 1000 RMB"],
  "soft_preferences": ["value_for_money", "low_effort"],
  "last_recommended_products": [],
  "rejected_products": [],
  "user_emotion": "neutral",
  "shopping_stage": "comparing"
}
```

## Effective Query

The Memory Layer builds an `effective_query` for downstream routing and retrieval.

Example:

```text
raw_query: mainly cares about value for money
effective_query: user is looking for a robot vacuum, budget around 1000 RMB, prioritizing value for money.
```

`raw_query` is preserved for trace/debug. `effective_query` is used by Supervisor and Retrieval.

## Selective Memory Injection

Do not pass full chat history to all Agents.

Each Agent receives a memory view:

```text
Supervisor View: task continuity, active category, shopping stage, emotion
Retrieval View: effective query, category, budget, ranking objective, filters
Recommendation View: preferences, rejected brands/products, last recommendations
Explanation View: user cares, selected facts, relation evidence summary
Critic View: hard constraints, user emotion, ranking objective, last query
```

This keeps Agent inputs smaller, less noisy, and easier to test.

## Context Compression

Default recent window: 6 messages.

When a session exceeds the window, the backend should call LLM to generate a structured summary:

```json
{
  "active_task": "buy robot vacuum",
  "stable_preferences": ["value_for_money", "low_effort"],
  "hard_constraints": ["budget around 1000 RMB"],
  "negative_feedback": ["does not want expensive options"],
  "latest_intent": "continue comparing better-value choices"
}
```

This summary is for Agents and should not be shown directly to normal users.

During compression, frontend should show:

```text
CONTEXT_COMPRESSION_RUNNING
```

## Non-Goals For V3 First Pass

- Do not introduce Mem0, Zep, Letta, or another heavy memory framework.
- Do not create a Memory Agent.
- Do not persist long-term user profiles.
- Do not display raw memory summaries to users.
- Do not solve context bugs with one-off prompt patches.
